# 🗄️ Дерекқорлар: ER-модельдеу, Нормализация, SQL және Тұтастық

Бұл құжат магистратура КТ *(M094)* профиль пәнінің маңызды бөлімі — **дерекқорлар** тақырыбын жүйелі түрде түсіндіреді.

---

## Ресми тест спецификациясы бойынша

`Дерекқор базасы.doc` файлы бойынша бір нұсқада **20 тапсырма** болады. Жалпы уақыт: **50 минут**, бір тапсырмаға орташа **2,5 минут**.

Тапсырмалар жабық формада беріледі: **бір немесе бірнеше дұрыс жауапты** таңдау керек. DB пәнінде жауап саны 1-3 аралығында болуы мүмкін. Толық дұрыс жауап - **2 балл**, бір қате - **1 балл**, екі немесе одан көп қате - **0 балл**.

Қиындық деңгейі бойынша үлес:

| Деңгей | Саны | Үлесі |
| ------ | ---- | ----- |
| A - жеңіл | 5 | 25% |
| B - орта | 9 | 45% |
| C - күрделі | 6 | 30% |

Ресми тақырыптар үлесі:

| Блок | Тақырыптар | Саны |
| ---- | ---------- | ---- |
| Реляциялық модель | реляциялық деректер моделі, реляциялық алгебра, реляциялық есептеулер, жиын операциялары | 2 |
| Тұтастық | домендік, сілтемелік, семантикалық тұтастық | 2 |
| Архитектура | файл-сервер, клиент-сервер, үш деңгейлі архитектура, SQL Server физикалық/логикалық деңгейі | 2 |
| ER-модель | нысандар, атрибуттар, ER-диаграмма элементтері | 1 |
| Кілттер және байланыстар | функционалды тәуелділік, әлеуетті/бастапқы/сыртқы кілт, байланыс түрлері | 2 |
| SQL | агрегаттық функциялар, фильтрлеу, іздеу, сұрыптау, топтастыру, DDL/DML, ішкі және байланысқан SQL | 8 |
| Жобалау және нормализация | концептуалдық/логикалық/физикалық жобалау, 0NF/1NF/2NF/3NF, денормализация | 3 |

PDF үлгісіндегі формат: сұрақтар A-H нұсқаларымен беріледі, SQL сұраныстарын оқу, ER/реляциялық модель терминдерін тану, ДҚБЖ ядросы мен транзакция қасиеттерін ажырату сияқты тапсырмалар кездеседі.

## ✅ 1. Дерекқорларды жобалау және ER-модельдеу

Бұл бөлімде дерекқордың құрылымын дұрыс құра алу қабілеті тексеріледі.

### 📌 ER-диаграмма (Entity–Relationship)

ER-модельдеу негізгі ұғымдарға сүйенеді:

* **Entity (E)** — *Субъект / Нысан*
* **Relationship (R)** — *Байланыс*

📝 Мысал сұрақ:

> ER-диаграммасындағы **E** әрпі нені білдіреді?

✅ Жауабы: **Entity — Субъект/Нысан**

---

### 🔗 Байланыс түрлері

| Түрі         | Белгіленуі | Мысал                     |
| ------------ | ---------- | ------------------------- |
| Бірдің-бірге | 1:1        | Бір адам — бір паспорт    |
| Бірдің-көпке | 1:M        | Бір мұғалім — көп студент |
| Көптің-көпке | M:N        | Көп студент — көп пән     |

---

### 🧩 Атрибуттар

* **Жай атрибуттар** — бөлінбейтін мән (мыс: Жасы)
* **Күрделі атрибуттар** — бірнеше бөлікке бөлінеді (мыс: Толық аты)
* **Кілттік атрибуттар** — объектіні бірегей анықтайды

---

## ✅ 2. Реляциялық модель және Нормализация

Бұл бөлім кестелер теориясын және деректер құрылымын дұрыс ұйымдастыруды қамтиды.

### 📍 Нормализация деңгейлері

Нормализация — артық деректерді азайтып, құрылымды дұрыс жасау процесі.

| Формалар | Атауы                     |
| -------- | ------------------------- |
| 1NF      | Бірінші нормальді форма   |
| 2NF      | Екінші нормальді форма    |
| 3NF      | Үшінші нормальді форма    |
| BCNF     | Бойс-Кодд нормальді форма |

📝 Мүмкін сұрақтар:

* «3NF шарты қандай?»
* «Денормализация не үшін қолданылады?»

---

### Термин

**Денормализация** — өнімділікті арттыру үшін кейде артық ақпаратты әдейі қайтару.

---

### 📌 Функционалдық тәуелділік

Жазу түрі:

> X → Y

Бұл нені білдіреді?

✅ **Y мәні X-ке функционалды тәуелді**

---

### 🔑 Кілттер

| Кілт түрі   | Анықтамасы                          |
| ----------- | ----------------------------------- |
| Primary Key | Кестедегі жазбаны бірегей анықтайды |
| Foreign Key | Басқа кестемен байланыс орнатады    |

📌 Бұл кілттер деректер тұтастығын сақтайды.

## ✅ 3. SQL (Structured Query Language) — Практикалық блок

Тестте нақты кесте беріліп, сұраныстың нәтижесін анықтау талап етіледі.

---

### 📍 Таңдау және сұрыптау

Негізгі операторлар:

* `SELECT`
* `FROM`
* `WHERE`
* `ORDER BY` (ASC/DESC)
* `LIMIT`

📝 Мысал:

```sql
SELECT ID
FROM Students
ORDER BY GPA DESC
LIMIT 3;
```

✅ GPA бойынша ең жоғары алғашқы 3 студентті табады.

---

### 📊 Агрегаттық функциялар және топтастыру

Функциялар:

* `SUM()`
* `AVG()`
* `MAX()`
* `MIN()`
* `COUNT()`

Операторлар:

* `GROUP BY`
* `HAVING`

---

### 🔁 Жиындармен жұмыс

| Оператор  | Қызметі               |
| --------- | --------------------- |
| UNION     | Нәтижелерді біріктіру |
| INTERSECT | Ортақ бөлікті табу    |
| EXCEPT    | Айырмашылығын шығару  |

---

### 🔗 Кестелерді біріктіру

JOIN түрлері:

* Inner Join
* Left Join

## ✅ 4. ДҚБЖ архитектурасы және Тұтастық

### Архитектура түрлері

| Түрі                   | Сипаттамасы                            |
| ---------------------- | -------------------------------------- |
| Файл-сервер            | Барлық дерек бір серверде файл ретінде |
| Клиент-сервер          | 2 деңгейлі жүйе                        |
| 3 деңгейлі архитектура | UI + логика + дерекқор бөлек           |

---

### Деректер тұтастығы (Integrity)

| Түрі                  | Анықтамасы                     |
| --------------------- | ------------------------------ |
| Домендік тұтастық     | Типі мен диапазоны дұрыс болуы |
| Сілтемелік тұтастық   | Foreign Key дұрыс байланысуы   |
| Семантикалық тұтастық | Бизнес логикаға сай болуы      |

---

### 🔒 Транзакциялар және ACID

ACID қасиеттері:

* **Atomicity** — толық орындалады немесе орындалмайды
* **Consistency** — дерекқор дұрыс күйде қалады
* **Isolation** — транзакциялар бір-біріне әсер етпейді
* **Durability** — нәтиже сақталып қалады
